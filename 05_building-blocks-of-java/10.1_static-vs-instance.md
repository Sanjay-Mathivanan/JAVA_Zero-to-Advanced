# Static vs Instance Members in Java

## Introduction

When learning Java, one common question is:

```text
When should I use static?
When should I use instance variables and methods?
```

The easiest way to understand this concept is:

```text
Instance  → Belongs to an Object

Static    → Belongs to a Class
```

---

# Real-World Example

Consider a college.

Every student has:

```text
Name
Age
Roll Number
```

But all students belong to the same:

```text
College Name
```

Example:

```text
Student 1 → Sanjay
Student 2 → Rahul
Student 3 → Arun

College → KGISL Institute of Technology
```

Notice:

```text
Student Name
```

changes for every student.

But:

```text
College Name
```

is common for everyone.

---

# Instance Variables

## Definition

Instance variables belong to objects.

Every object gets its own copy.

---

## Example

```java
class Student {

    String name;

}
```

Create Objects:

```java
Student s1 = new Student();
Student s2 = new Student();

s1.name = "Sanjay";
s2.name = "Rahul";
```

Output:

```java
System.out.println(s1.name);
System.out.println(s2.name);
```

```text
Sanjay
Rahul
```

---

## Memory View

```text
Student Object 1
---------------
name = Sanjay
---------------

Student Object 2
---------------
name = Rahul
---------------
```

Each object stores its own data.

---

# Static Variables

## Definition

Static variables belong to the class.

Only one copy exists.

---

## Example

```java
class Student {

    static String college =
            "KGISL";

}
```

Access:

```java
System.out.println(Student.college);
```

Output:

```text
KGISL
```

---

## Memory View

```text
Class Area
----------------
college = KGISL
----------------

Student Object 1

Student Object 2

Student Object 3
```

Only one copy is shared by all objects.

---

# Complete Example

```java
class Student {

    String name;

    static String college =
            "KGISL Institute of Technology";

}

public class Main {

    public static void main(String[] args) {

        Student s1 = new Student();
        Student s2 = new Student();

        s1.name = "Sanjay";
        s2.name = "Rahul";

        System.out.println(s1.name);
        System.out.println(s2.name);

        System.out.println(Student.college);
    }
}
```

---

## Output

```text
Sanjay
Rahul
KGISL Institute of Technology
```

---

# Instance Methods

## Definition

Instance methods belong to objects.

They require object creation.

---

## Example

```java
class Student {

    String name;

    void display() {

        System.out.println(name);

    }
}
```

Usage:

```java
Student s1 = new Student();

s1.name = "Sanjay";

s1.display();
```

Output:

```text
Sanjay
```

---

# Static Methods

## Definition

Static methods belong to the class.

No object is required.

---

## Example

```java
class Calculator {

    static int add(int a, int b) {

        return a + b;

    }
}
```

Usage:

```java
System.out.println(
        Calculator.add(10,20));
```

Output:

```text
30
```

---

# Why is main() Static?

Java starts program execution before creating any object.

Therefore:

```java
public static void main(String[] args)
```

must be static.

# Static Counter Example

One of the most common uses of static variables is counting objects.

```java
class Student {

    static int count = 0;

    public Student() {

        count++;

    }
}

public class Main {

    public static void main(String[] args) {

        new Student();
        new Student();
        new Student();

        System.out.println(
                Student.count);
    }
}
```

Output:

```text
3
```

Explanation:

```text
Object 1 Created → count = 1

Object 2 Created → count = 2

Object 3 Created → count = 3
```

Since count is static, all objects share the same variable.

# Static vs Instance

| Instance | Static |
|-----------|---------|
| Belongs to Object | Belongs to Class |
| Multiple Copies | One Copy |
| Requires Object | No Object Required |
| Stores Individual Data | Stores Shared Data |
| Access Using Object | Access Using Class Name |

# Interview Questions

## What is a Static Variable?

A variable shared by all objects of a class.

---

## What is an Instance Variable?

A variable that belongs to an individual object.

---

## Can Static Methods Access Instance Variables Directly?

No.

Because static methods do not have an object reference.

---

## Can Instance Methods Access Static Variables?

Yes.

---

## Why is main() Static?

Because Java needs to execute it before creating any object.

---

## How Many Copies of a Static Variable Exist?

```text
One
```

---

## How Many Copies of an Instance Variable Exist?

```text
One Per Object
```

---

# Key Takeaways

- Instance members belong to objects.
- Static members belong to classes.
- Every object gets its own instance variables.
- Static variables have only one shared copy.
- Static methods can be called without creating objects.
- Instance methods require objects.
- Use static for common data.
- Use instance for object-specific data.

---

# Conclusion

Static and Instance members are fundamental concepts in Java. Instance members represent data and behavior unique to each object, while static members represent data and behavior shared across all objects of a class. Understanding this distinction helps you write efficient, reusable, and memory-friendly Java applications.
